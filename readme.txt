Для запуска приложения различных компах с Linux или Unix используйте скрипт "start.sh", но для Helios надо запускать "start_for_helios.sh".
Это связано с тем, скрипт для Helios запускает определенную версию Java. ВАЖНО! Скрипт должен лежать в одном каталоге с jar файлом и файлом коллекции in.json.
Можно, конечно, сделать иначе, но для этого нужно изменять пути до jar файла и файла коллекции.
Но и тут может быть подвох, при попытке запуска скрипта, у вас может не хватать прав для этого. Сначала нужно решить эту проблему.
Решили? Тогда запускаем терминал, переходим в директорию с приложением, и запускаем скрипт: ./start.sh (или ./start_for_helios.sh).
Если все было проделанно правильно, то запустится само приложение.

Теперь немного о файле in.json.
Он был изначально заполнен данными, для примера, чтобы сразу можно было запустить программу. Когда же это вышло, его можно заполнять так,
как нужно вам. Завершите программу командой "exit", и можете заполнять файл на ваш вкус. Как только это сделали, можно заново запускать приложение, и оно
подгрузит уже ваши данные. Пример заполнения in.json:

{
  "products": [
    {
      "name": "name",
      "coordinates": {
        "x": 1.0,
        "y": 1
      },
      "price": 1.0,
      "partNumber": "partNumber",
      "manufactureCost": 1,
      "unitOfMeasure": "GRAMS",
      "manufacturer": {
        "name": "name",
        "annualTurnover": 1,
        "organizationType": "PUBLIC"
      }
    },

    .
    .
    .

    ]
}

name, partNumber - строки
x, price - дробные числа
y, manufactureCost, annualTurnover - целые числа
unitOfMeasure - принимает значения: MILLILITERS, METERS, MILLIGRAMS, GRAMS, SQUARE_METERS (ВАЖНО! Написано это должно быть Caps-lock)
organizationType - принимает значения: PUBLIC, GOVERNMENT, TRUST  (ВАЖНО! Написано это должно быть Caps-lock)


Итак, после заполнения коллекции в файле и запуска приложения, можно с ней теперь повзаимодействовать.
Введите help, он вам почти все расскажет!
